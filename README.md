# Viora — مهام وروابط ومشاريع مشتركة

موقع بسيط فيه سيكشنين:
- **المهام**: كل مشروع ليه مهامه، وأي مهمة تعملها صح بتتشطب.
- **الروابط**: تحفظ لينك وتكتبله وصف يفكرك ليه سيبته.

وفيه مشاركة مشاريع كاملة:
- كل مشروع ليه **فريق**. تقدر تبعت **رابط دعوة** لأي حد، أو تدعي حد بالاسم عن طريق **يوزرنيمه**.
- الشخص اللي بيتدعى باليوزرنيم بتظهرله دعوة يقدر **يقبلها أو يرفضها**.
- أي عضو يضيف/يعدّل/يخلّص/يحذف مهمة، كل أعضاء المشروع بيشوفوا التغيير على طول (لايف)، وبيظهر في **"النشاط الأخير"** أسفل قائمة المهام.
- كل مهمة وكل لينك ليهم **سجل خاص بيهم** — زرار "السجل" جوه كارت المهمة/اللينك نفسه بيوريك تسلسل اللي حصل فيها بالظبط (مين أضافها، مين عدّلها، إلخ) بترتيب زمني.
- لينك تأكيد الحساب اللي بيوصل على الإيميل بياخد اليوزر لصفحة تسجيل الدخول وبيقوله "تم تأكيد حسابك، سجّل دخولك دلوقتي".
- مفيش صفحة "رئيسية" منفصلة؛ أول ما تسجل دخول بتوصل على طول لتاب "المهام"، وجنبه تاب "الروابط".

مبني بـ Next.js + Tailwind، والداتا بيز Supabase (بما فيها الـ Realtime).

## 1. جهّز Supabase

1. افتح مشروعك في supabase.com (أو اعمل مشروع جديد).
2. روح على **SQL Editor** وشغّل محتوى ملف `supabase/schema.sql` اللي في المشروع ده (نسخ/لصق وRun). آمن تشغّله أكتر من مرة، وهيظبط كل الجداول والدوال والصلاحيات لوحده حتى لو كان عندك نسخة قديمة من قبل. الجداول اللي بيعملها:
   - `profiles` (اسم يوزر يونيك)
   - `projects`, `tasks`, `links`
   - `project_members` (أعضاء كل مشروع وحالتهم: مقبول/معلّق)
   - `invite_links` (روابط الدعوة)
   - `activity_log` (سجل نشاط المشروع + سجل كل مهمة لوحدها عبر `task_id`)
   - `link_activity_log` (سجل كل لينك لوحده)
3. من **Database → Replication** (أو **Table Editor** → أيقونة الـ Realtime)، فعّل الـ Realtime على جدولي `tasks` و `activity_log` عشان التحديثات تظهر لايف لكل أعضاء المشروع من غير ما يعملوا Refresh.
4. من **Authentication → Providers → Email**، سيب "Confirm email" شغالة (ده اللي بيخلي رسالة التأكيد تتبعت). لو حابب تقفلها، اليوزر هيقدر يسجّل دخول على طول من غير تأكيد إيميل.
5. من **Authentication → URL Configuration**:
   - حط **Site URL** = رابط موقعك (مثلاً `https://your-app.vercel.app` أو `http://localhost:3000` وقت التجربة).
   - ضيف في **Redirect URLs** الرابط: `.../auth/callback` (استبدل `...` برابط موقعك، أو حط `http://localhost:3000/**` وقت التجربة المحلية عشان يشتغل مع أي بورت/رابط).
   - ده اللي بيخلي لينك تأكيد الإيميل ياخد اليوزر لصفحة تسجيل الدخول بدل ما يفتحله حاجة تانية.
6. من **Project Settings → API** خد:
   - `Project URL`
   - `anon public key`

## 2. شغّل المشروع محليًا

```bash
npm install
cp .env.local.example .env.local
```

افتح `.env.local` وحط فيه القيم اللي جبتها من Supabase:

```
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=xxxxxxxx
```

بعدين:

```bash
npm run dev
```

افتح http://localhost:3000

## 3. الرفع على Vercel

1. ارفع المشروع على GitHub (repo جديد).
2. من Vercel، اعمل **Import Project** واختار الـ repo.
3. في **Environment Variables** ضيف نفس المتغيرين اللي في `.env.local`:
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
4. Deploy، وخلاص هيشتغل.

## ملاحظة عن الحماية (RLS) وتعدد المستخدمين

كل يوزر بيسجل حساب باسم مستخدم (username) لازم يكون يونيك — الجدول `profiles` فيه constraint بيمنع تكراره، وفي دالة `username_exists` بتتأكد إنه متاح قبل ما يخلّص التسجيل.

**الروابط** شخصية بحتة: كل يوزر بيشوف لينكاته هو بس، مش مرتبطة بمشروع ولا بتتشارك.

**المشاريع والمهام** بقت مشتركة: أي عضو "مقبول" في `project_members` بتاعة مشروع معيّن يقدر يشوف/يضيف/يعدّل مهام المشروع ده، حسب سياسات RLS. الكتابة في `project_members` نفسها (الدعوة، القبول، الرفض، الانضمام باللينك) بتتم كلها من خلال دوال SQL محددة (`invite_user_by_username`, `respond_to_invite`, `join_project_by_invite`, `get_or_create_invite_link`) بصلاحية `security definer`، مش عن طريق كتابة مباشرة في الجدول من الفرونت، عشان محدش يقدر يضيف نفسه لمشروع من غير دعوة أو لينك صحيح.

## البنية

```
app/
  layout.tsx           # الفونتات والـ RTL
  page.tsx             # الصفحة الرئيسية (تابز مهام/روابط + الدعوات المعلّقة، مفيش تاب "رئيسية")
  login/page.tsx        # تسجيل الدخول/حساب جديد + رسالة تأكيد الإيميل
  join/[token]/page.tsx # صفحة الانضمام لمشروع عن طريق رابط دعوة
  auth/callback/page.tsx# استقبال لينك تأكيد الإيميل وتوجيهه لصفحة الدخول
  globals.css           # تصميم الورقة والـ checkbox
components/
  TasksSection.tsx  # المشاريع + المهام (لايف)
  LinksSection.tsx  # الروابط والوصف
  TeamPanel.tsx      # فريق المشروع: أعضاء + دعوة بيوزرنيم + رابط دعوة
  ActivityFeed.tsx   # سجل نشاط المشروع كله (لايف)
  ItemHistory.tsx     # سجل خاص بمهمة أو لينك واحد بس، جوه كارته
  PendingInvites.tsx # بانر الدعوات المعلّقة اللي جالك من حد تاني
lib/
  supabase.ts        # الاتصال بالداتا بيز + الأنواع
supabase/
  schema.sql          # السكريبت اللي بتشغله في Supabase
```
